# 团队与技术栈

---

## 我们在做什么

我们负责 **前端互动类项目的完整交付**，从模板到上线。  
整体工作围绕三个核心目标展开：

- **稳定上线**
  - 确保项目在不同环境、设备、浏览器下都能正常运行  
- **可复用沉淀**
  - 将通用逻辑、结构、样式沉淀为模板或组件，减少重复开发  
- **高效率交付**
  - 在保证质量的前提下，快速完成需求并按时上线  

---

## 日常交付内容

### 互动玩法
- 抽奖 / 老虎机 / 转盘 / 合成 / 卡牌 / 点击类等 玩法
- 主要负责互动逻辑、动画表现与页面联动

### 域名相关页面
- 主域名页面、活动落地页
- 负责页面结构、样式统一及多尺寸适配

### 加载页（Loader）
- 页面或资源加载过程中的过渡页面
- 用于展示加载进度、优化首屏体验
- 与主流程解耦，通常可跨项目复用

### EndCard（EC）
- 活动结束后的展示页 / 引导页 / 跳转页
- 与主流程解耦，通常可跨项目复用

---

## 交付原则

- **稳定**
  - 多环境（测试 / 正式）、多设备、多浏览器下可正常运行
- **可复用**
  - 鼓励模板化、组件化，避免重复造轮子
- **高效率**
  - 明确流程与规范，减少返工和沟通成本

## 技术栈（默认基线）

> 以下为团队当前默认使用的技术栈，实际项目可能根据需求调整

### 基础技术
- **HTML / CSS / SCSS**
  - 页面结构、布局与样式规范
- **JavaScript**
  - 业务逻辑与交互实现

### 框架与构建
- **Vue2 / Vue3**
  - 项目主要使用 Vue 体系，根据项目情况选择版本
- **Vite**
  - 作为构建工具，用于提升开发和构建效率

### 动画方案
- **CSS Animation**
  - 简单动画、过渡效果、状态反馈
- **GSAP**
  - 复杂时间轴动画与高自由度动画控制

## 新人阶段目标（前 1～2 周）

在入职初期，希望你能够：

- ✅ **独立跑起项目**
  - 能完成本地启动、构建与基础调试
- ✅ **完成简单需求**
  - 按既有规范修改页面或互动逻辑
- ✅ **定位常见问题**
  - 能处理路径、资源、样式、动画相关问题
- ✅ **理解交付流程**
  - 明确项目从开发 → 构建 → 发布 → 上线的整体路径
