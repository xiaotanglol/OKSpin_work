# 开发环境与工具

本章节用于确保新同事能够 **正确、完整地搭建开发环境**。  
请 **严格按顺序完成**，部分步骤缺失将 **直接影响开发或发布流程**。

---

## 必装环境（必须完成）

### Node.js

- **统一使用 Node.js v16+ 建议v20**
- **强烈建议使用 nvm 管理 Node 版本**
  - 便于多项目切换
  - 避免版本冲突或误升级

> 不建议只全局安装一个 Node 版本

#### 验证
```bash
node -v
```

---

### Git（必须，重要）

⚠️ **Git 环境必须优先配置完成**  
否则在远程协作、居家办公、项目发布等场景下将无法正常工作。

---

#### 1️⃣ 安装 Git
- 通过 **Git 官方渠道**下载安装 Git for Windows

---

#### 2️⃣ 配置 SSH（必须）
- 按以下教程完成 SSH 配置：  
  https://blog.csdn.net/su_sf99/article/details/134554047

> SSH 用于拉取私有仓库、发布项目，是必备能力

---

#### 3️⃣ 拷贝 SSH 配置文件（必须）
- 将提供的 `config` 文件拷贝到 `.ssh` 目录

Windows 路径：
```
C:\Users\ASUS\.ssh
```

> ⚠️ 缺失该配置会导致仓库访问或发布异常

---

#### 4️⃣ 替换 Git 安装目录下的 `usr` 文件夹（必须）
- 用于使用项目内置的 `rsync`
- **项目发布流程会依赖该工具**

> ⚠️ 未替换将无法正常发布项目

---

#### 5️⃣ 替换 hosts 文件（必须）
- 路径：
```
C:\Windows\System32\drivers\etc
```

用途：
- 确保域名可正常访问
- 避免拉取 / 发布过程中的网络异常

---

#### 6️⃣ Git 客户端（可选）
- 可自行安装 **TortoiseGit**
- 作为 Git 的图形化操作工具使用

---

## 核心仓库说明（重点）

当前主要维护的仓库包括：

- **official_website**
  - 域名主仓库
  - 用于官方域名页面相关开发

- **okspin-template**
  - 较早期的互动玩法仓库
  - 主要用于历史项目维护

- **okspin-tml-loader**
  - 加载页（Loader）相关仓库
  - 负责资源加载与首屏体验

- **okspin-tml-v2**
  - 新一代主框架仓库
  - 覆盖内容包括：
    - 互动玩法
    - 域名 UI
    - EndCard（EC）

> 日常开发以 **okspin-tml-v2** 和 **official_website** 为主

## 开发工具

开发工具可按个人习惯选择：

- VS Code
- Cursor
- Trae
- Antigravity

---

### 必装 VS Code 插件（重要）

#### oks-tml-helper

- **内部自研插件**
- 默认仅支持 **VS Code**
- 其他开发工具 **插件市场无法直接搜索**
- 必须通过 **离线方式** 安装

##### 插件用途
- **域名项目** 、 **EndCard（EC）** 
- 运行
- 项目构建
- 项目上传

##### 安装方式（VS Code）
- 使用 VS Code 提前将 **插件离线保存到本地**
- 使用其他开发工具的 **“从 VSIX 安装插件”**
- 插件文件示例：
```
oks-tml-helper.vsix
```

##### 其他开发工具说明
- Cursor / Trae / Antigravity 如需使用该插件：
  - 请自行查阅对应工具的插件导入文档
  - 以插件功能可正常使用为准

⚠️ **域名与 EndCard 项目必须安装该插件**
