
# AI 开发工作流（团队标准）

> 面向团队成员的 AI 使用规范与开发流程说明

## 1. AI 使用安全规范

### 1.1 基本原则

AI 是开发辅助工具，而不是自动交付系统。

所有 AI 生成内容必须：

- 人工 Review
- 本地运行验证
- 符合团队编码规范
- 覆盖基本测试

禁止直接复制 AI 代码上线。

### 1.2 数据安全

允许输入：

- 脱敏后的技术代码
- UI 结构
- Mock 数据
- 抽象业务逻辑

禁止输入：

- 用户隐私数据
- 内部 API 地址
- Token / Key
- 商业机密逻辑

## 2. 团队 AI 开发标准

### 2.1 技术统一约束

所有 AI 生成代码必须符合：

- Vue3 + `<script setup>`
- Scoped CSS（组件作用域隔离）
- Composition API
- 语义化 HTML
- ESLint + Prettier 规范
- 禁止全局样式污染

### 2.2 标准 Prompt 结构

所有团队 Prompt 必须遵循统一结构：

```text
【角色】
【上下文】
【输入】
【约束】
【输出格式】
```

## 3. AI 标准开发流程

### 3.1 标准流程

需求 → Prompt → AI 生成 → Review → 测试 → PR

### 3.2 最佳实践

- 小步生成
- 快速验证
- Prompt 复用
- 持续优化

## 4. 新人培训与 Checklist

### 4.1 新人流程

1. 阅读本指南
2. 使用标准 Prompt 实战
3. 完成 Review 练习

### 4.2 开发 Checklist

> 每次提交代码前必须完成以下检查

开发前

- 需求是否理解清楚
- 是否选择了合适的 Prompt / 技术方案
- 是否确认数据已脱敏（涉及 AI 时）

开发中

- 是否遵守团队编码规范
- 是否避免无意义的重复代码
- AI 生成代码是否已阅读并理解
- 关键逻辑是否添加注释

开发后

- 本地是否可以正常运行
- 是否测试主要功能流程
- 是否检查边界情况
- 是否通过 ESLint / 格式化
- Git 提交信息是否清晰

提交前

- 是否删除调试代码与 console
- 是否更新相关文档
- 是否确认不影响其他模块
