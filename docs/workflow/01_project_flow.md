# 项目从需求到上线

> 目标：新人每天清楚 **先确认什么 → 再开发 → 怎么提测 → 怎么上线**

---

## 每日标准工作流程

### 1. 确认任务与需求

每天开始工作：

* 查看 **Admin 系统** 和 **飞书** 的任务排期
* **先和运营确认需求内容**

确认清楚：

* 功能范围
* 验收标准
* 截止时间

👉 **需求没确认清楚不要开始开发**

---

### 2. 查看设计稿

设计稿一般在 **Admin 中查看蓝湖链接**。

如果 Admin 没有：

👉 主动联系设计索要蓝湖链接

需要确认：

* UI 样式
* 动画形式
* 移动端适配
* 图片尺寸

---

### 3. 开发

开发阶段主要目标：

* 功能实现
* 动画效果
* 移动端适配

开发时：

* 使用分支开发
* 自测核心交互
* 确保 Console 无报错

---

### 4. 提测

开发完成后提测：

* 在 Admin 后台填写测试链接
* 上传项目截图/图片
* 修改项目状态为提测

同时：

👉 确认运营是否有修改意见

---

### 5. 修复与确认

根据运营/测试反馈：

* 按优先级修复问题
* 回归测试
* 确认问题关闭

---

### 6. 上线发布

上线前需要完成：

* 构建项目
* 验证线上环境
* 确认回滚方案

上线后立即检查：

* 页面加载是否正常
* 核心功能是否可用
* Console 是否报错

---

## 新人常见坑

* 没和运营确认需求就开始开发
* 设计稿没确认直接写代码
* 提测时忘记更新 Admin 信息
* 上线后没有线上验证

---

## 验收标准 ✅

你必须能在 3 分钟内讲清：

* 需求从哪里确认
* 设计稿在哪里
* 怎么开发和提测
* 怎么上线和回滚
